## v2 대비 변경분 — 4건 보강

### B-Z1 (BLOCKER — Codex 2회차) 모든 거부 = 403 통일
**사양 (`docs/03-PROMPTS.md` P1.2 + `docs/01-CLAUDE.md` P8):** 거부 응답은 "동일 메시지 + 동일 status code(403)".

v2의 응답 매트릭스가 404·422·500을 독립 상태로 둔 점 정정:
- **외부 응답**: validation 실패·middleware exception·unmatched route·internal server error 등 **모든 거부**는 `403 Forbidden` + UniformError 동일 body로 통일.
- **내부 로깅**: 원래 status code·exception 종류는 audit/log에 그대로 기록 (debugging 가능).
- **예외**: 정상 응답 200·201은 그대로 (정상은 다양해도 OK).
- 게이트 7 응답 매트릭스 명령어 정정: 정상(200/201) 1세트 + 모든 거부(외부 403, 내부 다양) 1세트의 2분포 비교만.

### M-Z1 (MAJOR — Codex 2회차) EML 헤더 fixed-width
**v2의 "정상=빈 헤더, 거부=값 헤더" 패턴 폐기.** 헤더 바이트 길이 누설.

- 모든 응답에 `X-Coesite-EML` 헤더 **fixed-width 32자리** (lowercase hex).
- 정상 응답: 32자리 dummy (예: `00000000000000000000000000000000`).
- 거부 응답: 실제 EML 시그널 hex (32자리로 정규화).
- 게이트에 추가: `header byte length` 동일성 검증 (헤더 set·각 header value 길이 모두 동일).

### S-G1 (Gemini SUGGESTION) Header Value Jitter 차단
M-Z1으로 흡수 (fixed-width 32자리가 jitter 차단).

### S-G2 (Gemini SUGGESTION) Compression Entropy
- 게이트 추가: gzip·deflate **압축 비활성화** 명시 (`compression: false`) 또는 **압축된 최종 바이트에 K-segment padding** 둘 중 선택. **압축 비활성화 추천** (단순·확실).
- NestJS `app.use(compression())` 등록 금지.
