## 결정사항 (단일 진실 — 모순 0)

### D1. 경로 (사양 일치)
`packages/api/src/common/oracle-prevention/` (`meta-layer/` 아님).

### D2. Padding 로직 단일화
`OraclePreventionService.padAndReject(res, finding?)` 단일 함수. 세 경로 모두 호출:
- TokenNormMiddleware: 변형 탐지 시 `await service.padAndReject(res, finding)` (middleware 안에서 직접 padding·response 종료)
- UniformErrorFilter: 일반 exception은 `service.padAndReject(res, {source:'exception'})` 호출
- OraclePreventionInterceptor: 성공 응답에 `service.padSuccess(res, body)` 호출
→ 세 경로 응답 분포 균일 보장.

### D3. 응답 분포 — 단 2종
- **정상 응답 (200/201)**: padding 적용된 body·고정 헤더 set
- **거부 응답 (외부 403)**: 모든 거부(403·404·422·500·middleware exception·unmatched route 등 원인 무관)가 외부엔 **403 + UniformError 동일 body**로 통일
- **내부 로깅**: 원래 status·exception type·stack은 audit/log에 그대로 기록 (debugging 가능)
- **404/422/500 외부 노출 금지**

### D4. EML 시그널 — fixed-width 32자리
- 모든 응답에 `X-Coesite-EML` 헤더, **32자리 lowercase hex** (정상·거부 동일 길이)
- 정상 응답: 32자리 dummy (예: `00000000000000000000000000000000`)
- 거부 응답: `eml(x,y)` 결과 hex (32자리 패딩)
- `y<=0`·NaN/Infinity는 P2 Fail-Closed

### D5. 압축 비활성화 (단일 정책)
- NestJS `app.use(compression())` 등록 금지
- `Content-Encoding` 헤더 응답에 부재 또는 `identity`만 PASS
- (압축 후 K-padding 옵션 폐기 — 단일 정책)

### D6. K-segment padding
- 모든 응답 body byte 크기 = 512 배수 (정상·거부 동일 규칙)
- 짧은 body는 padding(`\0` 또는 사양에 명시된 채움 문자)으로 K 배수까지

### D7. 시간 padding (최소 시간)
- 모든 응답 처리 시간 ≥ 200ms (configurable)
- 짧으면 setTimeout으로 padding
- 실제 처리 시간 측정 후 부족분만 추가

### D8. P1.1 BLOCKER 4건 흡수
- B1 AppModule 통합 ← app.module.ts 등록 + `APP_INTERCEPTOR`·`APP_FILTER` provider
- B2 Fixed-point 정규화 ← service.normalize() 변화 없을 때까지 루프 (최대 5회 cap·base64 팽창 256배 초과 fail-closed·재귀 depth 3 cap·각 회별 입력 크기 추적·cap 도달은 Finding 'convergence-failed' 메타데이터)
- B3 Malformed URL fail-closed ← `decodeURIComponent` throw 시 finding 분류·403 거부
- B4 대칭 padding ← D2 padding service 단일화로 흡수
