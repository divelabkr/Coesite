[v3 검토] **불통과 / 잔존 2건**

반영 확인:
- B-Z1 모든 거부=403: **반영됨**
- M-Z1 EML fixed-width: **반영됨**
- Gemini S-G1 header value jitter: **M-Z1로 흡수됨**
- Gemini S-G2 compression entropy: **부분 반영**

잔존:
1. **v2와 v3 충돌 문구 잔존**
   - v3가 403 통일·EML fixed-width로 정정했지만, “영향 맵·Phase A 양식은 v2 유지”라고 되어 있어 v2의 `404/422/500 매트릭스`, `정상 EML 빈 헤더` 문구가 다시 살아남을 수 있음.
   - 조치: v3에 “v2의 해당 문구는 폐기/대체”를 명시.

2. **압축 정책 수용 기준 모순**
   - S-G2는 “압축 비활성화 또는 압축 후 K-padding” 중 선택 가능이라 쓰지만, 게이트 18은 `Content-Encoding` 부재/`identity`만 통과시킴.
   - 조치: **압축 비활성화 필수**로 고정하거나, 압축 후 padding 선택지에 대한 별도 게이트를 추가.

요약: 4개 보강 의도는 맞지만, 구현자가 v2+v3를 함께 읽을 때 충돌할 수 있어 아직 통과로 보기 어렵습니다.
