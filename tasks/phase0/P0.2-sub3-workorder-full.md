## 결정사항 — codex review [P1] 즉시 처리

**D-P1 (cap_drop ALL 부팅 실패)**: `cap_drop: ALL`을 **narrow drop set**으로 변경:
```yaml
cap_drop:
  - NET_RAW
  - NET_ADMIN
  - SYS_ADMIN
  - SYS_PTRACE
  - SYS_MODULE
  - AUDIT_WRITE
  - DAC_READ_SEARCH
  - MAC_OVERRIDE
  - MAC_ADMIN
# CAP_CHOWN, CAP_SETUID, CAP_SETGID 등 entrypoint 필수 권한은 보존
```
또는 더 단순한 방식: `cap_drop: ALL` 유지 + `cap_add: [CHOWN, SETUID, SETGID, FOWNER, DAC_OVERRIDE]`. **Codex가 둘 중 안정적인 방식 선택**, 사유를 요약본에 기록.

## 결정사항 — MAJOR 처리

**M-Gemini 1 & 레드팀 M1 (Redis password 노출)**: `redis-cli -a` 인자 → 환경변수 방식:
- compose의 redis service에 `environment: { REDISCLI_AUTH: ${REDIS_PASSWORD} }` 추가
- healthcheck를 `redis-cli ping` 으로 단순화 (REDISCLI_AUTH가 자동 적용)
- requirepass는 `--requirepass ${REDIS_PASSWORD}` 인자 형식 유지하되 추후 `*_FILE` 방식 도입은 backlog

**레드팀 M2 (placeholder credential 무방어)**: 
- `infra/preflight.sh` 스크립트 신규: docker compose up 전 .env 파일 검증 (placeholder 패턴 `change-me-locally` 잔존 시 거부, 빈 password 거부, `0.0.0.0` bind 거부)
- README "시작" 순서에 `bash infra/preflight.sh && docker compose up -d` 형식으로 강제

**레드팀 M3 (loopback 우회 — host port 노출)**:
- 기본 compose에서 host port 매핑 **제거** (`ports:` 섹션 삭제. Postgres·Redis는 docker 네트워크 내부에서만 접근)
- `debug` profile 신규: `profiles: [debug]` + `ports: ["127.0.0.1:5432:5432"]` (Postgres), `"127.0.0.1:6379:6379"` (Redis)
- README: 일반 사용 `docker compose up -d`, 디버그 사용 `docker compose --profile debug up -d` 명시

**레드팀 M4 (volume 잔존)**:
- preflight.sh에 기존 volume 탐지 (`docker volume ls | grep coesite-` 결과 있으면 경고·확인 요구)
- README에 "시크릿 변경 또는 스키마 변경 시 `docker compose down -v` 필수" 강조

**레드팀 M5 (digest 갱신 절차)**:
- docker-compose.yml의 image 라인 위에 주석으로:
  - 조회일 (2026-05-22)
  - 조회 출처 (Docker Hub tags API URL)
  - 갱신 절차 (3개월마다 또는 보안 CVE 시 재조회, `docker pull` + `inspect`로 digest 갱신)
  - CVE scan 권고 (`trivy image postgres:16-alpine` 등)

**레드팀 M6 (healthcheck 깊이)**:
- 현재 healthcheck 유지 (인증·쿼리 검증 충분)
- 향후 P0.3 Prisma 시점에 "앱용 최소권한 계정 read/write 검증" 추가하라는 주석을 healthcheck 옆에 명시
- API 서비스 추가 시 `depends_on: { postgres: { condition: service_healthy } }` 주석 강화 (이미 sub-2에서 추가됨, 재확인)

**레드팀 M7 (README 동기화 누락)**:
- README에 `packages/api/.env` 생성 절차 명시 + `infra/.env`와의 credential 동기화 표 추가:

| infra/.env | packages/api/.env | 동기화 |
|---|---|---|
| POSTGRES_USER + POSTGRES_PASSWORD + POSTGRES_DB | DATABASE_URL 안에 같은 값 | ✅ 일치 |
| REDIS_PASSWORD | REDIS_URL의 password 부분 | ✅ 일치 |
