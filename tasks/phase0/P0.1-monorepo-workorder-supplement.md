## 1. 결정사항 (Claude가 확정 — 그대로 따를 것)

### D1. Build 도구
- **`tsc`만 사용.** 번들러(esbuild·tsup·webpack 등) 도입 금지. Phase 6 SDK 빌드에서 별도 검토.
- 각 패키지 `package.json` script: `"build": "tsc -b"` (project references) 또는 `"build": "tsc"` 단순 호출. 패키지별 선택 자유.

### D2. NestJS skel 깊이 (packages/api)
- **두 파일만:** `src/main.ts` + `src/app.module.ts`.
- `app.module.ts`: `@Module({})` 빈 모듈.
- `main.ts`: `NestFactory.create(AppModule).then(app => app.listen(3000))` 수준의 최소 부팅 코드. 포트는 상수 OK(나중에 ConfigService로 교체).
- Controller·Service·DTO 일체 만들지 않는다. (P0.2~ 이후 별도 task에서 추가.)
- NestJS peer deps (reflect-metadata, rxjs 등)는 `dependencies`에 포함.

### D3. Path alias 전략
- **tsconfig `paths` 설정하지 않는다.**
- 패키지 간 import는 `pnpm workspace:*` 의존 선언만으로 해결 (node_modules 심볼릭 링크).
- 예: `packages/api/src/main.ts`에서 (지금은 안 쓰지만 미래에) `import { Foo } from '@coesite/types'` 형태로 자연스럽게 동작해야 함.
- `tsconfig.base.json`의 `moduleResolution`은 `"node"` 또는 `"nodenext"` 중 NestJS 호환되는 쪽 선택. (NestJS 10 권장: `"node"`.)

### D4. Build 산출물·gitignore
- 각 패키지 build 산출물 = `dist/`.
- 루트 `.gitignore`에 다음 항목 필수: `node_modules/`, `dist/`, `*.log`, `.env`, `.env.local`, `coverage/`, `.turbo/`, `.next/`, `.DS_Store`, `pnpm-debug.log*`.
- `.gitignore`에 `docs/codex-log/`는 **포함 안 함** (요약본은 커밋 대상).
