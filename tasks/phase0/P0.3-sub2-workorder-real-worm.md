### B2 — TRUNCATE 차단 (BLOCKER B2)
WORM 5개 테이블 모두에 추가:
```sql
CREATE TRIGGER worm_<table>_truncate
  BEFORE TRUNCATE ON "<Table>"
  FOR EACH STATEMENT
  EXECUTE FUNCTION prevent_truncate();
```
별도 함수:
```sql
CREATE OR REPLACE FUNCTION prevent_truncate() RETURNS trigger AS $$
BEGIN
  RAISE EXCEPTION 'WORM violation: TRUNCATE on % is forbidden', TG_TABLE_NAME
    USING ERRCODE = 'P0002';
END; $$ LANGUAGE plpgsql;
```

### B3 — Chain Integrity (BLOCKER B3 + MAJOR hash unique)
schema.prisma의 WORM 5개 모델에:
- `hash` → `@unique` 추가 (중복 hash 차단)
- `(agentId, prevHash)` 복합 unique (AuditLog만 — 다른 모델은 prevHash unique만 또는 적절히)

새 migration `<TS+2>_chain_integrity/migration.sql`:
```sql
ALTER TABLE "AuditLog" ADD CONSTRAINT audit_hash_unique UNIQUE (hash);
ALTER TABLE "AuditLog" ADD CONSTRAINT audit_prev_unique UNIQUE ("agentId", "prevHash");
-- WormLog, AdminActionLog, DmsTriggerLog, ProofBundle 동일 패턴

ALTER TABLE "AuditLog" ADD CONSTRAINT audit_prev_format CHECK ("prevHash" ~ '^[a-f0-9]{64}$' OR "prevHash" = 'GENESIS');
```

### B4 — INSERT 검증 트리거 (BLOCKER B4)
WORM 5개 테이블에 BEFORE INSERT 트리거:
```sql
CREATE OR REPLACE FUNCTION verify_chain_insert() RETURNS trigger AS $$
DECLARE
  prior_exists boolean;
  genesis_count int;
BEGIN
  -- prevHash가 GENESIS면 → 해당 테이블에 GENESIS가 하나뿐인지 확인
  IF NEW."prevHash" = 'GENESIS' THEN
    EXECUTE format('SELECT COUNT(*) FROM %I WHERE "prevHash" = ''GENESIS''', TG_TABLE_NAME)
      INTO genesis_count;
    IF genesis_count > 0 THEN
      RAISE EXCEPTION 'Chain violation: duplicate GENESIS on %', TG_TABLE_NAME
        USING ERRCODE = 'P0003';
    END IF;
  ELSE
    -- prevHash가 GENESIS 아니면 → 해당 hash가 이미 존재해야 함
    EXECUTE format('SELECT EXISTS(SELECT 1 FROM %I WHERE hash = $1)', TG_TABLE_NAME)
      INTO prior_exists USING NEW."prevHash";
    IF NOT prior_exists THEN
      RAISE EXCEPTION 'Chain violation: prevHash % not found on %', NEW."prevHash", TG_TABLE_NAME
        USING ERRCODE = 'P0004';
    END IF;
  END IF;
  RETURN NEW;
END; $$ LANGUAGE plpgsql;

CREATE TRIGGER chain_audit_log BEFORE INSERT ON "AuditLog"
  FOR EACH ROW EXECUTE FUNCTION verify_chain_insert();
-- 5개 테이블 동일 패턴
```
