## 결정사항

**N1 — 명명 통일**
- `infra/.env.example`: `DATABASE_URL` 단독 변수 제거 → `OWNER_DATABASE_URL` (migration용) + `RUNTIME_DATABASE_URL` (앱용, PASSWORD NULL이라 Phase 1 전엔 미사용).
- `packages/api/.env.example`: 동일 — `OWNER_DATABASE_URL` + `RUNTIME_DATABASE_URL` 두 변수. **Phase 1 전엔 `DATABASE_URL=OWNER_DATABASE_URL`** 별칭 권장 주석.

**N2 — preflight 강제 검증**
- `infra/preflight.sh`에 추가:
  - `OWNER_DATABASE_URL`과 `RUNTIME_DATABASE_URL`의 host·port·database 이름 일치 확인.
  - 불일치 시 `exit 3` ("URL consistency violation").
  - 두 URL의 user 부분이 같으면 경고 (owner=runtime은 의도된 misuse 위험).

**N3 — README 가이드**
- "Phase 1 진입 전 runtime role 사용 가이드" 섹션 추가:
  ```
  ## Phase 1 이전 runtime role 사용 가이드
  현재 coesite_runtime은 PASSWORD NULL로 비활성 상태입니다.
  - Phase 1 application 완성 전: DATABASE_URL=$OWNER_DATABASE_URL 사용
  - Phase 1 진입 후 활성화 절차:
    1. ALTER ROLE coesite_runtime PASSWORD :secret_from_kms;
    2. .env의 RUNTIME_DATABASE_URL을 실제 값으로 교체
    3. Prisma client multi-datasource 구성 (별도 task)
  ```
