2. D-P1 cap_drop 처리 방안  
선택: **narrow drop set**  
이유: DB 이미지 entrypoint에 필요한 `CHOWN`, `SETUID`, `SETGID`, `FOWNER`, `DAC_OVERRIDE` 계열 권한을 보존해 부팅 실패 위험을 줄이고, 위험도가 큰 capability만 명시적으로 제거하는 쪽이 안정적이다.

3. MAJOR 7 처리 방안 한 줄씩  
M1: Redis healthcheck는 `REDISCLI_AUTH` 환경변수 기반 `redis-cli ping`으로 변경한다.  
M2: `infra/preflight.sh`를 추가해 placeholder, 빈 password, `0.0.0.0` bind를 차단한다.  
M3: 기본 compose host `ports`를 제거하고 `debug` profile에서만 `127.0.0.1` 바인딩을 허용한다.  
M4: preflight에서 기존 `coesite-` volume을 탐지해 경고·확인을 요구하고 README에 `down -v`를 강조한다.  
M5: image digest 주석에 조회일, 출처, 갱신 절차, CVE scan 권고를 추가한다.  
M6: healthcheck는 유지하되 P0.3 Prisma 시점 최소권한 계정 검증 TODO 주석을 붙인다.  
M7: README에 `packages/api/.env` 생성 절차와 `infra/.env` credential 동기화 표를 추가한다.

4. MINOR 6 처리 방안 한 줄씩  
N1: `.gitignore`에 `**/*.pem`, `**/*.key`, `**/*.p12`, `**/*.pfx`, `**/*.jks`를 추가한다.  
N2: README에 `pnpm -r build`, `pnpm --filter @coesite/api dev/test` 예시를 추가한다.  
N3: `.gitignore`에 `**/.env.*`와 `!**/.env.example` 예외를 추가한다.  
N4: compose의 `PGPASSWORD` 환경변수를 제거하고 healthcheck 안에서만 인라인으로 사용한다.  
N5: `restart: unless-stopped`를 `restart: on-failure:5`로 변경한다.  
N6: README에 `docker compose down -v`, 로컬 env 삭제 절차를 추가한다.  
참고: container_name enumeration, tmpfs, resource limit은 이번 범위 밖 backlog로 둔다.

5. preflight.sh 스크립트 설계 요약  
검증 항목: `infra/.env`, `packages/api/.env`의 placeholder 잔존, 빈 password, `0.0.0.0` bind, 기존 `coesite-` volume 존재 여부.  
종료 코드: `0` 통과, `1` hard fail, `2` volume 경고에서 사용자가 진행 거부.
