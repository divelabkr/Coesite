## [P0.3 sub-2] Codex 이해 확인 (Phase A)

1. 영향 맵 대조  
수정 대상: `prisma/schema.prisma`, 기존 `prisma/migrations/V_002_worm_triggers.sql` 삭제, `.gitignore`  
신규 대상: `prisma/migrations/<TS+1>_worm_triggers/migration.sql`, `<TS+2>_chain_integrity/migration.sql`, `<TS+3>_role_separation/migration.sql`  
영향 맵 밖 파일 수정 금지.

2. B1~B8 처리 방안 한 줄씩  
B1: 기존 flat SQL migration을 삭제하고 init 이후 timestamped migration 디렉토리로 재작성.  
B2: WORM 5개 테이블에 `prevent_truncate()` 및 `BEFORE TRUNCATE` 트리거 추가.  
B3: WORM 5개 모델/테이블에 hash unique, prevHash chain unique, hash/prevHash format CHECK 추가.  
B4: WORM 5개 테이블에 `verify_chain_insert()` 및 `BEFORE INSERT` 체인 검증 트리거 추가.  
B5: `coesite_runtime` role을 만들고 WORM은 INSERT/SELECT만, UPDATE/DELETE/TRUNCATE는 금지.  
B6: WORM payload에 PII 경고 주석을 추가하고 README에 Phase 1 전 PII·암호화 결정 항목 추가.  
B7: `Agent` 모델에 nullable `lastDecayAt DateTime?` 추가.  
B8: `.gitignore`에 `.pnpm-cache/` 추가.

7. 위험 3개

- 기존 데이터가 hash/prevHash unique 또는 CHECK 조건을 위반하면 chain integrity migration 실패 가능.  
- role separation은 placeholder password와 운영 secret/KMS 적용이 분리되어 있어 Phase 1+에서 실제 runtime 연결 누락 위험.
