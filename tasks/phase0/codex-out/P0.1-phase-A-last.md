## [P0.1] Codex 이해 확인 보고

### 1. 범위 복창 (내 말로 요약)
- 만들 패키지 4개: `@coesite/types`, `@coesite/utils`, `@coesite/api`, `@coesite/sdk`
- 사용할 도구·버전: pnpm, TypeScript 5.4 strict, Node 22, NestJS 10
- 별칭 정책: `@coesite/types`, `@coesite/utils`를 실제 워크스페이스 패키지와 일치시키고 `workspace:*`로 연결

### 2. 절대 금지 사항 복창
- 보안 금지어 목록 원문에 정의된 모든 항목과 변형 표기를 변수명·주석·문서·커밋 메시지·출력에 사용하지 않음
- P1: 판단형 AI/ML 프레임워크 및 예측 호출 금지
- P6: `packages/api`는 향후 `@coesite/types`, `@coesite/utils` 의존 선언 필요
- P7: 존재하지 않는 import·경로·alias 참조 금지
- F12: 실제 시크릿 값 하드코딩 금지
- 이번 실행에서 구현하지 않는 항목: 파일 생성·수정·삭제, 의존성 설치, 빌드, 테스트, 타입체크, 스캔 실행

### 3. 실행 계획 (Phase B에서 할 일을 순서대로)
1. 루트 `pnpm-workspace.yaml` 작성
2. 루트 `package.json`에 `dev`, `build`, `test`, `scan`, `lint` scripts 구성
3. 루트 `tsconfig.base.json` 작성
4. 루트 `.gitignore`, `.editorconfig`, `.nvmrc` 작성
5. `packages/types` 패키지 구성
6. `packages/utils` 패키지 구성
7. `packages/api` 패키지 구성 및 NestJS 10 최소 진입점 작성
8. `packages/sdk` 패키지 skel 구성
9. `packages/utils/test/sanity.test.ts`에 vitest skel 테스트 1건 작성
10. 의존성 설치 및 lock 파일 생성
11. 빌드·타입체크·테스트·보안 금지어 스캔 검증
12. 마지막: `docs/codex-log/2026-05-21-P0.1-monorepo.md` 요약본 작성

### 4. 의문점·확인 필요 사항
- 없음

### 5. 예상 위험
- 의존성 설치 실패 가능성: 네트워크, pnpm 설정, 패키지 버전 해석 문제 가능
- TypeScript path alias 충돌 가능성: 루트 base 설정과 각 패키지 `tsconfig.json`의 `paths`, `outDir`, `rootDir` 정합성 확인 필요
- 기타: NestJS 최소 skel에서 필요한 peer/runtime 의존성 누락 가능성, 보안 금지어가 의존성 파일이나 생성 산출물에 포함될 가능성

### 6. 검증 계획 (Phase B 완료 직전)
- `pnpm install` 성공 여부 확인
- `pnpm -r build`로 전체 패키지 빌드 확인
- `npx tsc --noEmit`로 TypeScript 오류 0건 확인
- `npx vitest run`으로 skel 테스트 1건 통과 확인
- `rg` 기반 수동 스캔으로 보안 금지어 0건 확인
