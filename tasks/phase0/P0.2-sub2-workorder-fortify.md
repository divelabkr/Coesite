## 결정사항 — MAJOR 처리

**M-codex review 1 (.pnpm-store 119MB)**: 위 B2 처리에 포함. 추가로 git에 이미 들어간 추적 캐시는 없음(.pnpm-store는 줄곧 untracked) — 확인만.

**M-codex review 2 (env_file 객체 형식)**: `infra/docker-compose.yml`의 `env_file`을 **단순 문자열 형식**으로 변경:
```yaml
env_file:
  - ./.env
```
객체 형식(`path:`/`required:`) 사용 금지. 호환성 우선.

**M-Gemini 1 (README 부족)**: `README.md`에 추가:
- "사전 요구사항" 섹션: Node 22, pnpm 10, Docker (옵션 — 로컬 DB만 필요 시)
- "프로젝트 시작" 섹션: `pnpm install` → `pnpm -r build` → `pnpm test`
- "세션 시작 의무" 섹션: 01-CLAUDE.md §1 참조 명시 (헌법 준수 전파)
- 검증: `pnpm test`로 vitest 실행 방법

**M-Gemini 2 (depends_on race)**: `docker-compose.yml`에 향후 `api` 서비스 추가될 위치에 주석으로 명시:
```yaml
# NOTE: P0.3+ api 서비스 추가 시
#   depends_on:
#     postgres: { condition: service_healthy }
#     redis: { condition: service_healthy }
```

**M-레드팀 M1·M2 (Redis 인증 없음 + port 공개)**: 
- Redis: 개발용으로 인증 토큰 추가. `infra/.env.example`에 `REDIS_PASSWORD=change-me-locally` 추가. `docker-compose.yml`의 redis command에 `--requirepass "${REDIS_PASSWORD}"` 추가.
- 호스트 port는 명시적 `127.0.0.1:5432:5432`, `127.0.0.1:6379:6379` bind (외부 인터페이스 차단).

**M-레드팀 M3 (volume 잔존)**: README에 명시:
- `docker compose down`은 volume 보존 (개발 데이터 유지)
- `docker compose down -v`는 volume 삭제 (PII·시크릿 제거)
- 종료 시 의도 명확히 안내. **시크릿 변경 후 반드시 `down -v` 권고** 1줄 추가.

**M-레드팀 M4 (digest pin)**: image를 digest로 핀:
```yaml
postgres:
  image: postgres:16-alpine@sha256:<현재 latest의 digest>
redis:
  image: redis:7-alpine@sha256:<현재 latest의 digest>
```
- Codex가 `docker pull postgres:16-alpine && docker inspect --format='{{index .RepoDigests 0}}'`로 digest 조회 (호스트 docker 부재 시 NOT_RUN 표시하고 작업지시 변경 보류, 또는 Docker Hub API로 조회).

**M-레드팀 M5 (JWT replay)**: `packages/api/.env.example`의 `JWT_SECRET` 값을 더 명확한 placeholder로:
```
JWT_SECRET=__REPLACE_WITH_RANDOM_32+_BYTES_FROM_KMS__
```
+ README "시크릿 생성" 절차 (`openssl rand -hex 32` 예시).

**M-레드팀 M6 (healthcheck false healthy)**: 
- Postgres healthcheck를 `pg_isready` → `psql -U $POSTGRES_USER -d $POSTGRES_DB -c "SELECT 1"`로 변경 (인증·쿼리 검증).
- Redis healthcheck를 `redis-cli ping` → `redis-cli -a $REDIS_PASSWORD ping` 또는 `ACL WHOAMI` (인증 검증).
