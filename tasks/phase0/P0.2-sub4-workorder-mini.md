## 결정사항 (3건)

**D1 — README quick start 사용성 (codex review P2-1)**:
`cp .env.example → preflight → docker compose up` 사이에 명시적 단계 추가:
```
1. cp infra/.env.example infra/.env
2. ⚠️ infra/.env 편집 — POSTGRES_PASSWORD, REDIS_PASSWORD 등 모든 placeholder를 실제 값으로 교체
3. cp packages/api/.env.example packages/api/.env
4. ⚠️ packages/api/.env 편집 — DATABASE_URL, REDIS_URL의 password를 infra/.env와 동기화, JWT_SECRET을 openssl rand -hex 32 결과로 교체
5. pnpm db:up   # preflight 자동 실행 → docker compose up
```

**D2 — preflight JWT placeholder 차단 (codex review P2-2)**:
`infra/preflight.sh`에 추가 패턴:
- `JWT_SECRET=__REPLACE_WITH_RANDOM_32+_BYTES_FROM_KMS__` 거부
- `JWT_SECRET=change-me*` 거부
- `JWT_SECRET=` (빈값) 거부
- `JWT_SECRET` 길이 32자 미만 경고 (32+ 권장)

**D3 — preflight 강제력 (레드팀 B1)**:
`package.json` 루트 scripts에 추가:
```json
"db:up": "bash infra/preflight.sh && docker compose --env-file ./infra/.env -f ./infra/docker-compose.yml up -d",
"db:down": "docker compose --env-file ./infra/.env -f ./infra/docker-compose.yml down",
"db:reset": "docker compose --env-file ./infra/.env -f ./infra/docker-compose.yml down -v && rm -f infra/.env packages/api/.env"
```
README에서 `docker compose up` 직접 호출 라인 모두 제거, `pnpm db:up`만 안내.
