## 게이트 (7개, 모두 PASS 또는 명확한 NOT_RUN 보고)

1. `docker compose --env-file ./infra/.env.example -f ./infra/docker-compose.yml config` (문법)
2. 임시 `.env` 복사 → `docker compose up -d` → 30초 대기 → `docker compose ps` 모두 healthy
3. `docker exec coesite-postgres psql -U coesite -d coesite_dev -c "SELECT 1"` → `1`
4. `docker exec coesite-redis redis-cli ping` → `PONG`
5. `docker compose down` exit 0 (임시 .env 삭제)
6. Mountain 키워드 `rg --fixed-strings` 변경 산출물 대상 0건
7. 시크릿 패턴 (AWS_·AKIA·-----BEGIN·real-looking password) 변경 산출물 대상 0건

호스트 Docker 부재 시 2~5는 NOT_RUN 정직 보고 (P2 Fail-Closed).

## 레드팀 관점 (Phase C에서 적용 — 핵심 8개)
1. Postgres trust 인증 폴백 위험
2. host port가 외부 인터페이스 노출 (`127.0.0.1:` bind 필요?)
3. 컨테이너 root 실행·non-root user 옵션
4. `.env.example` placeholder가 진짜 placeholder인지 (실수로 실제 패턴 남음?)
5. 볼륨 권한·백업 시 시크릿 leak 경로
6. `depends_on` 누락으로 NestJS API race (P0.3 영향)
7. image digest pin 부재 (공급망)
8. README의 내부 도메인·IP·인증 흐름 leak
