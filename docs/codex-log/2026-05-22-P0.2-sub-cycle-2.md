# P0.2 sub-cycle 2 Phase B

## 무엇을 했나

P0.2 sub-cycle 2의 fortify 지시를 적용했다. 변경 범위는 `.gitignore`, Docker Compose, env example, README로 제한했다.

- 모든 디렉터리의 `.env`, `.env.local`과 `.pnpm-store/`를 ignore 대상으로 강화했다.
- Postgres/Redis compose 설정을 loopback bind, digest pin, 인증 healthcheck, 최소 권한 옵션 중심으로 보강했다.
- Redis 개발용 password placeholder와 API env example의 JWT placeholder를 강화했다.
- README에 사전 요구사항, 시작/검증 명령, 세션 시작 의무, secret 생성, volume 삭제 기준, 운영 보안 주의사항, 후속 task 메모를 추가했다.

## 왜

기존 설정은 로컬 개발 편의 중심이어서 다음 위험이 남아 있었다.

- 루트 `.env`만 보호되어 하위 디렉터리 env 파일이 실수로 노출될 수 있었다.
- Redis 인증이 없고 DB 포트가 설정값에 따라 외부 인터페이스로 열릴 여지가 있었다.
- Docker image tag가 digest로 고정되지 않아 재현성과 공급망 검증이 약했다.
- healthcheck가 실제 인증/쿼리 성공을 충분히 확인하지 않았다.
- volume 보존/삭제 기준, secret 생성/교체 기준, 운영 환경 주의사항이 README에 부족했다.

## 변경 파일

- `.gitignore`
  - `.env`, `.env.local`을 `**/.env`, `**/.env.local`로 강화했다.
  - `.pnpm-store/`를 ignore 대상으로 추가했다.
- `infra/docker-compose.yml`
  - `env_file`을 단순 문자열 형식으로 변경했다.
  - `postgres:16-alpine`, `redis:7-alpine`을 Docker Hub 태그 API로 확인한 digest에 pin했다.
  - Postgres/Redis host port를 `127.0.0.1`로 고정했다.
  - Redis에 `--requirepass`를 추가했다.
  - Postgres healthcheck를 `psql ... SELECT 1`로 바꿨다.
  - Redis healthcheck를 password 포함 `redis-cli ... ping`으로 바꿨다.
  - 두 서비스에 `security_opt: no-new-privileges:true`, `cap_drop: ALL`을 추가했다.
  - 향후 API 서비스 추가 시 healthy dependency를 쓰라는 주석을 추가했다.
- `infra/.env.example`
  - `REDIS_PASSWORD=change-me-locally`를 추가했다.
- `packages/api/.env.example`
  - 컨테이너 외부/내부 DB URL 기준 주석을 추가했다.
  - Redis URL에 password placeholder를 반영했다.
  - `JWT_SECRET` placeholder를 KMS/32바이트 이상 랜덤값 요구 형태로 바꿨다.
- `README.md`
  - Node/pnpm/Docker 요구사항, `pnpm install`, `pnpm -r build`, `pnpm test` 흐름을 추가했다.
  - `01-CLAUDE.md` §1 세션 시작 의무를 명시했다.
  - Redis password, loopback bind 유지, secret 생성, volume 보존/삭제, secret 변경 후 `down -v`, initdb args 재초기화 조건을 문서화했다.
  - 프로덕션 secrets manager, container/volume 이름 무작위화, DevContainer/WORM/debug profile 후속 task를 명시했다.
