## 무엇을 했나

P1.3 M-1 AllowList + GTED 보안 게이트를 추가했다.

- `AllowListPolicyService`: 정책 provider 로드, HMAC-SHA256 서명 검증, 정책 만료/미래 발행/빈 allowList/과대 목록 fail-closed, TTL 캐시.
- `GtedService`: bounded graph-based edit distance. 키보드 인접 치환, 시각 유사 치환, 인접 transposition을 가중치로 계산한다.
- `M1GateService`: 정확 매칭은 `PASS`, 근접 변형은 `OBSERVE` + `SIREN` handoff, 미등록/오류/정책 불가/입력 과대는 `BLOCK`.
- `M1Guard`: 요청 후보 토큰을 추출하고 `m1GtedMetadata`를 누적한다. 차단 시 Nest `ForbiddenException`을 던져 기존 `UniformErrorFilter`가 균일 응답으로 처리할 수 있게 했다.
- `AllowListGtedModule`: AppModule에 import했지만 아직 전역 Guard로 적용하지 않았다.

## 왜

TokenNorm 다음 단계에서 "정규화된 토큰이 허용 정책에 있는가"를 검증해야 한다. 동시에 오타·근접 변형은 단순 unknown과 달리 공격 신호일 수 있으므로 SIREN 단계로 넘길 수 있게 표시해야 한다.

## 리스크 / 미해결

- Prisma `Policy` 테이블과 Redis TTL 캐시는 아직 실제 어댑터로 연결하지 않았다. P1.3의 안전핵은 provider 인터페이스와 검증 로직으로 구현했다.
- `M1Guard`는 AppModule에 모듈만 등록되어 있고 전역 적용은 하지 않았다. SIREN 없이 근접 변형을 통과시키면 P6 결합 의존 원칙상 위험하므로 P1.6 통합에서 적용해야 한다.
- GTED 비용 함수는 보수적인 bounded rule이다. P1.4/P1.5 이후 공격 corpus가 생기면 threshold를 별도 정책으로 조정해야 한다.
