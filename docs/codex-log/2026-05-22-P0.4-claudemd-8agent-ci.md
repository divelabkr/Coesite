## 리스크 / 미해결
- `git mv`를 실제 명령으로 시도했지만 `.git/index.lock` 생성 권한이 ACL에서 거부되어 실패했다. 이후 파일 시스템 이동으로 작업을 진행했다. 현재 `git status`는 이동을 staged rename으로 표시하지 않고 삭제+신규 파일로 표시한다. 커밋 전 사람이 `.git` 권한이 있는 환경에서 rename/stage를 확인해야 한다.
- `scripts/scan-principles.sh`는 기존 내용 유지 원칙에 맞춰 이동만 했다. 기본 `SCAN_DIR`가 `src`라 현재 monorepo의 `packages/` 전체를 직접 스캔하지 않는 한계가 남아 있다.
- `scan-principles.sh` 결과에서 P8/P9/P10은 Phase 이후 필수 항목으로 warning이지만 exit 0이다.
- YAML 검증을 위해 `yaml-lint`를 임시 실행했다. 홈 캐시 권한 문제로 `npm_config_cache`를 워크스페이스 내부로 지정해 실행했고, 추적 대상 임시 파일은 남기지 않았다. `.pnpm-cache` 일부는 권한 문제로 삭제 실패했지만 `.gitignore` 대상이다.
- git commit, git push는 실행하지 않았다.
