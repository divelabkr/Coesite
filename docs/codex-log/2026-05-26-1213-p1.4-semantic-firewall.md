## 무엇을 했나

P1.4 SemanticFirewall + MirrorModel을 추가했다.

- `SemanticFirewallService`: rule-only 의미 카테고리 검사, 위험 점수 계산, `PASS`/`OBSERVE`/`BLOCK` 산출.
- `MirrorModelService`: SemanticFirewall 결과로 기대 응답 형태를 만들고 실제 응답의 control/handoff/leak pattern을 비교.
- 입력 방어: 최대 depth, 최대 string byte, 최대 string field count를 두고 초과 시 fail-closed.
- 증거 방어: finding에는 원문 매칭 텍스트를 저장하지 않고 32 hex hash만 저장.
- SIREN 연동 준비: 중간 위험은 `OBSERVE` + `SIREN` handoff로 표시.

## 왜

P1.3이 allowList와 거리 기반 토큰 검사를 담당한다면, P1.4는 의미 패턴 기반으로 role override, prompt leak, tool injection, data exfiltration 등 요청 의도를 룰로 분류해야 한다. MirrorModel은 실제 응답이 기대 control을 벗어나는 순간 fail-closed로 잡기 위한 비교층이다.
