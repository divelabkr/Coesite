# P0.2 sub-cycle 3 Phase B

## 무엇을 했나

P0.2 sub-3 Phase B 결정사항 중 D-P1, MAJOR 7건, MINOR 6건을 적용했다. Docker가 로컬에 없어 Docker 실행 게이트 1~5는 NOT_RUN으로 처리했고, 정적 게이트 6~11 및 회귀 게이트 12는 통과했다.

## 왜

- `cap_drop: ALL`은 DB 이미지 entrypoint에 필요한 권한까지 제거할 수 있어 narrow drop set으로 바꿨다.
- Redis healthcheck의 password CLI 인자 노출을 줄이기 위해 `REDISCLI_AUTH` 환경변수 방식으로 바꿨다.
- 기본 DB 포트 노출은 제거하고, 필요할 때만 `debug` profile의 loopback 프록시로 접근하도록 분리했다.
- placeholder credential, 빈 password, `0.0.0.0` bind, stale volume 사용을 사전 점검하는 `infra/preflight.sh`를 추가했다.
- 기존 문서 일부에 Security Wall Tier 1 용어가 직접 남아 있어 정책 문서 참조 방식으로 일반화했다.

## 리스크 / 미해결

- Docker가 없어 실제 compose config/up/healthcheck/down 동작은 검증하지 못했다.
- `debug` profile은 Compose가 서비스 내부 `ports`만 profile 처리할 수 없어 별도 loopback 프록시 서비스로 구현했다. 실제 컨테이너 동작 검증은 Docker 환경에서 필요하다.
- Postgres/Redis에 명시한 non-root `user`는 workorder 요구사항대로 반영했지만, named volume 권한 문제는 실제 Docker 실행 전까지 확인되지 않았다.
- Redis ACL, `*_FILE` secrets, tmpfs/resource limit, 운영 container/volume namespace, Config 추상화는 후속 phase backlog로 남겼다.
