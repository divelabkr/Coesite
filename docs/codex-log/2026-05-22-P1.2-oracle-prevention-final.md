# P1.2 OraclePrevention final Phase B

## 무엇을 했나
- P1.2 OraclePrevention 모듈을 추가하고 AppModule에 전역 interceptor/filter로 연결했다.
- 성공 응답과 거부 응답 모두 512 byte K-segment JSON body, 동일 고정 헤더, `X-Coesite-EML` 32자리 hex 형식, 최소 200ms 시간 padding을 적용했다.
- TokenNorm을 fixed-point 정규화로 바꾸고 malformed URL, depth cap, size/expansion cap을 fail-closed finding으로 처리했다.
- TokenNorm middleware의 직접 `res.end()` 경로를 OraclePreventionService 호출로 대체했다.
- e2e 스크립트와 NestJS runtime e2e, 응답 분포 부하 e2e를 추가했다.

## 왜
- P1.1 blocker였던 AppModule 미통합, fixed-point 미흡, malformed URL 허용, 거부/정상 padding 비대칭 문제를 P1.2에서 흡수해야 했다.
- 외부 관찰자가 status/body/header/size/timing 차이로 내부 거부 원인을 역추론하지 못하게 응답 형태를 통일해야 했다.
- 204/301/302 및 404/422/500 계열 거부가 외부에 원래 status로 노출되지 않게 해야 했다.
