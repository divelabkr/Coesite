# P0.3 Prisma Schema

## 무엇을 했나
- P0.3 Phase B D1~D5 범위의 Prisma schema v1을 구현했다.
- 8개 모델을 추가했고, WORM 대상 5개 모델에는 `prevHash` NOT NULL, `hash`/`createdAt` 인덱스, `@@map`, Prisma 주석을 반영했다.
- init migration SQL과 WORM UPDATE/DELETE 차단 트리거 SQL을 추가했다.
- 루트 Prisma scripts와 API 패키지 Prisma 의존성을 추가했다.

## 왜
- P0.3 워크오더가 Postgres/Prisma 기반 증거 로그 모델과 WORM 차단 구조를 요구했다.
- `pnpm prisma:generate`가 루트 script에서 실행되므로, 루트에도 Prisma CLI와 기본 output용 `@prisma/client`가 필요했다.
- API 패키지에서 Prisma Client import 타입 검증을 하기 위해 간이 타입 import 파일을 추가했다.

## 변경 파일
- `prisma/schema.prisma`
  - `Agent`, `Session`, `Policy`, `AuditLog`, `WormLog`, `AdminActionLog`, `DmsTriggerLog`, `ProofBundle` 모델 추가.
  - `SessionStatus`, `WormLogLevel` enum 추가.
  - datasource/generator를 워크오더 지정값으로 설정.
- `prisma/migrations/20260522000000_init/migration.sql`
  - 8개 테이블, enum, 인덱스, FK 생성 SQL 추가.
- `prisma/migrations/V_002_worm_triggers.sql`
  - 5개 WORM 테이블의 BEFORE UPDATE OR DELETE 트리거 추가.
- `package.json`
  - `prisma:generate`, `prisma:migrate`, `prisma:reset` scripts 추가.
  - 루트 Prisma CLI 실행과 기본 Prisma Client output 생성을 위해 `prisma`, `@prisma/client` 추가.
- `packages/api/package.json`
  - `@prisma/client`, `prisma` 의존성 추가.
- `packages/api/src/prisma-client.check.ts`
  - Prisma Client 타입 import 검증용 파일 추가.
- `pnpm-lock.yaml`
  - Prisma 의존성 해석 결과 반영.
- `docs/codex-log/2026-05-22-P0.3-prisma-schema.md`
  - 본 작업 요약본 추가.

## 검증 결과
- Gate 1 PASS: `pnpm install`, `pnpm prisma:generate`
- Gate 2 PASS: `pnpm dlx prisma@6.19.3 validate --schema=prisma/schema.prisma`
  - 참고: unpinned `pnpm dlx prisma validate --schema=prisma/schema.prisma`는 최신 Prisma 7.8.0을 받아 D4의 datasource `url` 형식과 충돌했다. 워크오더 D4를 유지하기 위해 프로젝트 버전 6.19.3으로 고정해 검증했다.
  - `DATABASE_URL`은 더미 로컬 URL을 환경변수로 주입했다. 실제 DB 접속은 하지 않았다.
- Gate 3 PASS: `pnpm exec prisma format --schema=prisma/schema.prisma`
- Gate 4 NOT_RUN: Docker CLI 부재. `docker --version`이 명령 인식 실패.
- Gate 5 NOT_RUN: Docker CLI 부재로 Postgres 컨테이너 기반 UPDATE/DELETE 차단 검증 미실행.
- Gate 6 PASS: `pnpm exec tsc --noEmit -p packages/api`
- Gate 7 PASS: 구현 표면(`prisma`, `packages`, `package.json`) 기준 Tier 1 보안 키워드 fixed/variant scan 0건.
- Gate 8 PASS: `pnpm -r build`, `pnpm test`

## 리스크 / 미해결
- WORM 트리거 SQL은 작성했지만 Docker/Postgres 부재로 실제 DB 적용과 P0001 차단 동작은 검증하지 못했다.
- `prisma/migrations/V_002_worm_triggers.sql`는 워크오더 지정 경로 그대로 만들었다. Prisma migrate의 표준 migration 폴더 구조가 아니므로 실제 적용 절차는 별도 확인이 필요하다.
- INSERT 시 `prevHash` 체인 무결성 검증은 이번 D1~D5 범위에 포함되지 않아 구현하지 않았다.
- `pnpm dlx` 재검증을 위해 워크스페이스 내부에 `.pnpm-home`, `.pnpm-cache`, `.pnpm-store` 임시 디렉터리가 생성됐다. 삭제 명령은 실행 정책에 차단되어 남아 있다.
