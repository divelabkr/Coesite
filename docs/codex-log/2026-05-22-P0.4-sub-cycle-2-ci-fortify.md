# P0.4 sub-2 Phase B - CI fortify

## 무엇을 했나

P0.4 sub-2 Phase B 지시 13건과 codex review P2 1건을 적용했다. 결론적으로 CI 강제력, 보안 스캔 allowlist, 헌법 drift 검사, agent spec 정합, CODEOWNERS/branch protection 문서, 경로 표기 정리를 반영했고 12개 게이트는 모두 통과했다.

## 왜

기존 상태는 원칙 스캔 기본 경로가 모노레포 구조와 맞지 않았고, 일부 체크가 warning 또는 skipped 상태로 끝났다. 보안 스캔은 과도한 제외 범위 때문에 작업 문서와 기준 문서가 실제 검사 범위 밖에 있었고, 루트 헌법 파일과 docs 사본의 drift를 CI에서 막는 장치도 없었다.

## 변경 파일

- `scripts/scan-principles.sh`: 기본 스캔 경로를 `packages`로 변경, P1 고급 패턴 추가, P6 의존성 선언 검증 추가, P8/P9/P10 미구현 항목을 INFO로 분리하고 검증 가능한 위험 패턴은 실패로 유지.
- `scripts/security-wall.sh`: 제외 범위를 축소하고 `.security-wall-allowlist` 기반 glob allowlist를 추가, regex 변형 탐지를 확장.
- `.security-wall-allowlist`: 작업지시와 작업 로그 파일 glob 등록.
- `.github/workflows/ci.yml`: checkout credential persistence 비활성화, `SCAN_DIR=packages` 명시, 헌법 drift 및 agent spec 검사 job 추가, `security-wall` job을 `type-check`에만 의존하도록 변경.
- `.github/CODEOWNERS`: 전체 경로 owner 지정.
- `.github/branch-protection.md`: main 브랜치 보호 설정 가이드 작성.
- `.claude/agents/gate-enforcer.md`: handoff 조건을 docs spec과 일치시킴.
- `.claude/agents/test-sentinel.md`: coverage 기준을 docs spec과 일치시킴.
- `CLAUDE.md`, `docs/01-CLAUDE.md`: 세션 시작 경로를 `docs/` 명시 형식으로 수정하고 drift 비교 가능 상태 유지.
- `docs/04-SECURITY-WALL.md`: regex 변형 문서 기준 갱신.
- `.gitignore`: `tasks/security-wall.log` 예외 추가.
- `README.md`: Actions SHA pinning 후속 작업 메모 추가.
- `docs/00-MASTER-PLAN.md`: 보안 스캔 실패를 만든 제한 등급 표현 2곳을 안전한 일반 표현으로 치환.

## 리스크 / 미해결

- P8/P9/P10 런타임 구현은 아직 Phase 1+ 범위다. 이번 작업은 P0에서 검증 가능한 위험 패턴과 CI 강제력만 강화했다.
- Actions SHA pinning은 적용하지 않았다. README와 branch protection guide에 후속 작업으로 남겼다.
- YAML 검증을 위해 `npx yaml-lint`를 workspace npm cache로 실행했다. 프로젝트 의존성에는 추가하지 않았다.
