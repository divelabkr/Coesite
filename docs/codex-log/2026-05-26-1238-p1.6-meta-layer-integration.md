## 무엇을 했나

P1.6 MetaLayer 통합을 추가했다.

- `MetaLayerService`: M1 → SemanticFirewall/MirrorModel → Polyintent/SIREN → Honeypot 순서로 평가해 `ALLOW` / `DECEIVE` / `DENY`를 산출.
- `MetaLayerGuard`: 요청에서 후보 토큰, payload, request/session/source metadata를 추출하고 평가 결과를 `metaLayerMetadata`에 부착. `DENY`는 `ForbiddenException`으로 fail-closed.
- `MetaLayerModule`: P1.3/P1.4/P1.5 모듈을 통합 import/export.
- AppModule은 개별 P1.3~P1.5 모듈 대신 `MetaLayerModule`을 import.

## 왜

P1.3~P1.5가 각각 독립 보안층이면 실제 게이트가 아니다. P1.6은 정상/약한 의심/강한 공격/허니팟/다중 우회 흐름을 하나의 결론으로 합쳐 Phase 2가 그 결과를 출처/속도/세션/사람 승인 경계와 연결할 수 있게 한다.
