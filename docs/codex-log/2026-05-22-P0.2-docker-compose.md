# P0.2 Docker Compose 로컬 환경

## 무엇을 했나

결론: Docker Compose 로컬 인프라 산출물을 추가했다.

- `infra/docker-compose.yml`에 Postgres 16 Alpine, Redis 7 Alpine 서비스를 정의했다.
- `infra/.env.example`과 `packages/api/.env.example`에 로컬 개발용 placeholder 값을 추가했다.
- `README.md`에 로컬 환경 시작/확인/종료 절차를 추가했다.
- 자체 codex review 호출은 하지 않았다.

## 왜

P0.2 Phase B 작업지시의 결정사항 1~6을 반영하기 위해서다.

- Postgres와 Redis를 로컬에서 반복 실행 가능한 Compose 서비스로 고정해야 했다.
- 실제 시크릿을 넣지 않고, 사람이 로컬에서 `.env`를 복사해 교체하는 구조가 필요했다.
- 실행 절차는 README에 남겨 다음 작업자가 같은 순서로 재현할 수 있게 했다.

## 변경 파일

- `infra/docker-compose.yml`
  - `postgres:16-alpine` 서비스 추가.
  - 컨테이너명 `coesite-postgres`, 호스트 포트 `5432`, 볼륨 `coesite-postgres-data`, `pg_isready` healthcheck 설정.
  - `.env` 참조와 UTF-8 관련 환경값, password 인증용 initdb args 설정.
  - `redis:7-alpine` 서비스 추가.
  - 컨테이너명 `coesite-redis`, 호스트 포트 `6379`, 볼륨 `coesite-redis-data`, `redis-cli ping` healthcheck, `--appendonly yes` 설정.
- `infra/.env.example`
  - `COMPOSE_PROJECT_NAME=coesite`.
  - Postgres/Redis 로컬 연결값과 `POSTGRES_PASSWORD=change-me-locally` placeholder 추가.
- `packages/api/.env.example`
  - `DATABASE_URL`, `REDIS_URL`, `JWT_SECRET`, `JWT_EXPIRES_IN`, `NODE_ENV`, `PORT` placeholder 추가.
- `README.md`
  - "로컬 환경 시작" 섹션 신규 작성.
  - Docker 사전요구, `.env` 복사, `docker compose up -d`, `ps healthy` 확인, `down`, 임시 `.env` 삭제 절차 추가.
- `docs/codex-log/2026-05-22-P0.2-docker-compose.md`
  - 이번 작업 요약본 작성.

## 검증 결과

- 게이트 1: NOT_RUN
  - 명령: `docker compose --env-file ./infra/.env.example -f ./infra/docker-compose.yml config`
  - 결과: 호스트에서 `docker` 명령을 찾을 수 없어 Compose config 검증을 실행하지 못했다.
- 게이트 2: NOT_RUN
  - 사유: 호스트 Docker 부재. 임시 `.env` 복사, `docker compose up -d`, 30초 대기, `docker compose ps` healthy 확인을 실행하지 않았다.
- 게이트 3: NOT_RUN
  - 사유: 호스트 Docker 부재. `docker exec coesite-postgres psql -U coesite -d coesite_dev -c "SELECT 1"`을 실행하지 않았다.
- 게이트 4: NOT_RUN
  - 사유: 호스트 Docker 부재. `docker exec coesite-redis redis-cli ping`을 실행하지 않았다.
- 게이트 5: NOT_RUN
  - 사유: 호스트 Docker 부재. `docker compose down`과 임시 `.env` 삭제 절차를 실행하지 않았다. 이번 작업 중 실제 `infra/.env` 파일은 만들지 않았다.
- 게이트 6: PASS
  - 명령: 금지 키워드 고정 문자열 스캔을 `infra/docker-compose.yml`, `infra/.env.example`, `packages/api/.env.example`, `README.md` 대상으로 실행.
  - 결과: 0건.
- 게이트 7: PASS
  - 명령: 클라우드 환경변수 접두어, 액세스키 접두어, PEM 헤더 고정 문자열 스캔을 `infra/docker-compose.yml`, `infra/.env.example`, `packages/api/.env.example`, `README.md` 대상으로 실행.
  - 결과: 0건.
  - 명령: password/secret/token/key 계열 값 중 placeholder가 아닌 긴 값을 찾는 정규식 스캔을 같은 대상으로 실행.
  - 결과: 0건.

요약: 2/7 PASS, 5 NOT_RUN.

## 리스크 / 미해결

- Docker CLI가 없어 Compose 문법, 컨테이너 healthcheck, Postgres 접속, Redis ping, `down` 정리를 실제로 검증하지 못했다.
- `POSTGRES_INITDB_ARGS`의 password 인증 설정은 Docker 실행 환경에서 확인이 필요하다.
- 이미지 digest pin, non-root 실행, 로컬 포트 bind 정책 등은 Phase C 레드팀 검토 범위로 남겼다.
